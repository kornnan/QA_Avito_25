# Функциональные E2E тест-кейсы для таск-трекера

Базовый URL: `https://avito-tech-internship-psi.vercel.app/issues`

## Обозначения

- **P1** — критичный сценарий, блокирующий работу пользователя
- **P2** — важный сценарий
- **P3** — вспомогательный / нефункциональный

---

## T1. Создание задачи (позитивный сценарий)

**ID:** T1  
**Приоритет:** P1  
**Цель:** Проверить, что пользователь может создать новую задачу с валидными данными.

**Предусловия:**

- Пользователь открыл страницу списка задач `/issues`.

**Шаги:**

1. Нажать на кнопку **"Create issue"** (или аналогичную по смыслу).
2. Ввести валидный заголовок задачи (например, `Test issue title`).
3. Ввести валидное описание задачи (например, `Test issue description`), если поле доступно.
4. Выбрать проект/тип/приоритет (если доступны, заполнить обязательные поля).
5. Нажать кнопку **"Submit" / "Create"**.
6. Дождаться возврата к списку задач (или появления новой карточки в списке).

**Ожидаемый результат:**

- Созданная задача отображается в списке задач.
- Заголовок задачи соответствует введённому значению.
- При клике на созданную задачу открывается карточка с корректными данными (заголовок, описание и т.д.).

---

## T2. Создание задачи — валидация обязательных полей

**ID:** T2  
**Приоритет:** P1  
**Цель:** Проверить, что задача не создаётся без заполнения обязательных полей.

**Предусловия:**

- Пользователь открыл страницу списка задач `/issues`.

**Шаги:**

1. Нажать на кнопку **"Create issue"**.
2. Оставить поле заголовка пустым.
3. Нажать кнопку **"Submit" / "Create"**.

**Ожидаемый результат:**

- Появляется сообщение об ошибке/валидации для обязательного поля (заголовка).
- Задача не создаётся и не появляется в списке.
- Кнопка создания/форма остаётся доступной для исправления.

---

## T3. Открытие карточки задачи из списка

**ID:** T3  
**Приоритет:** P1  
**Цель:** Проверить, что при клике по задаче из списка открывается корректная карточка задачи.

**Предусловия:**

- На странице `/issues` есть как минимум одна задача.
- Пользователь видит список задач.

**Шаги:**

1. Запомнить заголовок любой задачи в списке (например, первой).
2. Кликнуть по строке/карточке этой задачи.
3. Дождаться загрузки страницы карточки задачи.

**Ожидаемый результат:**

- Открывается страница карточки задачи.
- Заголовок задачи на карточке совпадает с заголовком, отображённым в списке.
- Отображаются основные атрибуты задачи (описание, статус, проект, исполнитель и т.п., если предусмотрены).

---

## T4. Открытие карточки задачи через прямой URL

**ID:** T4  
**Приоритет:** P2  
**Цель:** Проверить, что карточку задачи можно открыть по прямой ссылке.

**Предусловия:**

- Известен ID уже существующей задачи.
- Пользователь открывает URL вида `/issues/<id>`.

**Шаги:**

1. Перейти по прямому URL существующей задачи.
2. Дождаться загрузки страницы.

**Ожидаемый результат:**

- Открывается карточка задачи с корректными данными.
- Отображаемый заголовок и другой контент соответствуют ожидаемым для данного ID.

---

## T5. Поиск задачи по точному заголовку

**ID:** T5  
**Приоритет:** P1  
**Цель:** Проверить, что поиск корректно находит задачу по её заголовку.

**Предусловия:**

- На странице `/issues` существует задача с уникальным заголовком `Unique search test issue`.

**Шаги:**

1. Убедиться, что задача `Unique search test issue` присутствует в списке (создать при необходимости).
2. В поле поиска ввести `Unique search test issue`.
3. Нажать кнопку **"Search"** или дождаться автоматического срабатывания фильтра (в зависимости от реализации).
4. Дождаться обновления списка задач.

**Ожидаемый результат:**

- В списке отображается задача с заголовком `Unique search test issue`.
- Другие задачи, не соответствующие запросу, скрыты/отфильтрованы.

---

## T6. Поиск задачи по частичному совпадению

**ID:** T6  
**Приоритет:** P2  
**Цель:** Проверить, что поиск работает по части заголовка.

**Предусловия:**

- Существует задача `Search partial match issue`.

**Шаги:**

1. Ввести в поле поиска подстроку `partial match`.
2. Запустить поиск.
3. Дождаться обновления списка.

**Ожидаемый результат:**

- В результатах поиска присутствует задача `Search partial match issue`.
- Список не пустой (при наличии задач, подходящих под фильтр).

---

## T7. Поиск задачи — отсутствие результатов

**ID:** T7  
**Приоритет:** P2  
**Цель:** Проверить поведение интерфейса, когда по запросу нет задач.

**Предусловия:**

- Открыта страница `/issues`.

**Шаги:**

1. Ввести в поле поиска заведомо несуществующий запрос, например `__no_such_issue_12345__`.
2. Запустить поиск.

**Ожидаемый результат:**

- Список задач пуст.
- Отображается сообщение пользователю (например, "Ничего не найдено"), либо визуальный индикатор пустого результата.

---

## T8. Переход на доску проекта из списка задач

**ID:** T8  
**Приоритет:** P1  
**Цель:** Проверить, что пользователь может перейти на доску проекта из списка задач.

**Предусловия:**

- Пользователь находится на `/issues`.
- В интерфейсе есть ссылка/кнопка для перехода на доску проекта (например, "Project board").

**Шаги:**

1. Нажать на ссылку/кнопку **"Project board"** или аналогичную.
2. Дождаться загрузки страницы доски.

**Ожидаемый результат:**

- Открывается страница доски проекта.
- URL меняется на `/board` или другой, предопределённый для доски.
- На доске отображаются колонки/статусы и задачи.

---

## T9. Переход на доску проекта из карточки задачи

**ID:** T9  
**Приоритет:** P2  
**Цель:** Проверить, что переход на доску проекта возможен из карточки задачи.

**Предусловия:**

- Открыта карточка существующей задачи.

**Шаги:**

1. Найти и нажать ссылку/кнопку для перехода на доску проекта (например, "Back to board" или "View on board").
2. Дождаться загрузки страницы доски.

**Ожидаемый результат:**

- Открывается доска проекта.
- На доске видна задача, с карточки которой был сделан переход (по заголовку/ID).

---

## T10. Перевод задачи по статусам на доске

**ID:** T10  
**Приоритет:** P1  
**Цель:** Проверить, что можно менять статус задачи (перетаскивать между колонками/выбирать статус).

**Предусловия:**

- Открыта доска проекта.
- Есть задача в начальном статусе (например, `To Do`).

**Шаги:**

1. Найти задачу в колонке `To Do`.
2. Перетащить её в колонку `In Progress` (или изменить статус через UI).
3. Дождаться обновления UI.
4. Обновить страницу (F5/refresh).
5. Убедиться, что задача по-прежнему в колонке `In Progress`.

**Ожидаемый результат:**

- Статус задачи визуально меняется в интерфейсе.
- После перезагрузки страницы статус сохраняется.

---

## T11. Некорректный прямой URL карточки задачи

**ID:** T11  
**Приоритет:** P3  
**Цель:** Проверить поведение приложения при открытии несуществующей задачи по прямой ссылке.

**Предусловия:**

- Пользователь открывает URL `/issues/99999999` (несуществующий ID).

**Шаги:**

1. Перейти по URL `/issues/99999999`.
2. Дождаться загрузки страницы.

**Ожидаемый результат:**

- Пользователь видит понятное сообщение об ошибке (например, "Issue not found") или редирект на список задач.
- Не происходит бесконечной загрузки или падения приложения.
